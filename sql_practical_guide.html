SQL Practical Guide

A complete reference for Database Management lectures.

Focusing on class examples: CREATE, SELECT, INSERT, LIKE, ALTER, CHECK, and DELETE.

1. CREATE DATABASE & USE

Explanation: These commands are the starting point. CREATE DATABASE provisions a new empty database. USE tells the SQL engine that all subsequent commands should be executed inside that specific database.

Setup Phase Context Switching
-- Create a new database named 'asos12' CREATE DATABASE asos12; -- Select the database to run queries in it USE asos12;

2. CREATE TABLE & PRIMARY KEY

Explanation: CREATE TABLE defines the structure of your data. You list the columns and their data types (like int for numbers, varchar(20) for text up to 20 characters). The PRIMARY KEY limits a column to only contain unique, non-null values, acting as the main identifier for the row.

Defining Architecture Data Types: INT, VARCHAR
-- Create a table named 'akar' with four columns CREATE TABLE akar ( id int PRIMARY KEY, -- Unique identifier, cannot be null name varchar(20), -- String up to 20 characters city varchar(20), age int );

3. INSERT INTO

Explanation: INSERT INTO adds new rows of data to a table. You can insert data without specifying column names (if you provide a value for every single column in the correct order), or you can explicitly define the columns you are inserting into.

Use Case A: Implicit Column Insertion (Single Row)

Provide values for all columns exactly as they are ordered in the table.

INSERT INTO akar VALUES (5, 'rean', 'dhuk', 22); INSERT INTO akar VALUES (1, 'ahmad', 'hawler', 20);
Use Case B: Explicit Column Insertion (Multiple Rows at Once)

Specify the exact columns (e.g., dbo.akar indicates the schema 'dbo' and table 'akar'). You can chain multiple rows via comma separation to save lines.

INSERT INTO dbo.akar (id, name, city, age) VALUES (2, 'layla', 'dhuk', 26), (3, 'dara', 'hawler', 24);

4. SELECT & WHERE

Explanation: SELECT * grabs all columns from a table. The WHERE clause acts as a filter, allowing you to establish conditions. Only rows matching the condition are returned.

Use Case A: Retrieve Everything
SELECT * FROM akar;
Use Case B: Exact Match Filtering

Used to find specific records using the equals (=) operator.

-- Find all details for 'ahmad' SELECT * FROM akar WHERE name = 'ahmad'; -- Find all records from the city 'dhuk' SELECT * FROM akar WHERE city = 'dhuk';

5. LIKE Operator (Pattern Matching)

Explanation: Used inside a WHERE clause to search for a specified pattern in a column. The percent sign % represents zero, one, or multiple characters. Extremely useful for search functionalities.

Use Case A: Starting With a Letter
-- Finds any name starting with 'a' (e.g., 'ahmad', 'ali', 'aso') SELECT * FROM akar WHERE name LIKE 'a%';
Use Case B: Containing a Letter Anywhere
-- Finds any name with letter 'a' anywhere inside (e.g., 'rean', 'dara', 'layla') SELECT * FROM akar WHERE name LIKE '%a%';
Use Case C: Ending With a Letter (Extra Context)
-- Finds any name ending with 'a' (e.g., 'dara', 'layla') SELECT * FROM akar WHERE name LIKE '%a';

6. ALTER TABLE & CHECK Constraint

Explanation: ALTER TABLE changes an existing table's structure after it has been created without losing data. ADD CONSTRAINT adds rules to the table. A CHECK constraint ensures that all values in a column satisfy a specific condition.

Data Integrity Logical Operators (AND / OR)
-- Modifies the 'akar' table. Adds a rule named 'chk_age'. -- Now, the database will REJECT any insert or update where age is NOT between 1 and 99. ALTER TABLE akar ADD CONSTRAINT chk_age CHECK (age > 0 and age < 100);

7. DELETE & UPDATE (Important Additions)

Explanation: Both DELETE and UPDATE are DML (Data Manipulation Language) commands. They change existing data. ⚠️ Critical Rule: Always use a WHERE clause with these, otherwise, you might affect the entire table accidentally!

Use Case A: Deleting One Specific Row

To safely delete a single row, always target the Primary Key (in this case, id), because it is guaranteed to be unique.

-- Deletes ONLY the record where ID is 5 ('rean') DELETE FROM akar WHERE id = 5;
Use Case B: Deleting Multiple Rows Based on a Condition
-- Deletes everyone who lives in 'hawler' DELETE FROM akar WHERE city = 'hawler';
Use Case C: Modifying Specific Data (Not deleting the entire row)

If you want to clear out or change specific data in a row without deleting the row entirely, use UPDATE.

-- Change 'ahmad's city from 'hawler' to 'dhuk' UPDATE akar SET city = 'dhuk' WHERE id = 1; -- Nullify a value (make it empty) instead of deleting the row UPDATE akar SET age = NULL WHERE name = 'layla';
Use Case D: Deleting ALL Rows (Be careful!)
-- Deletes all rows but keeps the table structure intact DELETE FROM akar;
Pro Tip for Exams: Always pay attention to strings vs. numbers. Strings (like names or cities) must be wrapped in single quotes ('dhuk'), while numbers (like id or age) are kept unquoted (22). When asked to write a query, ensure your spelling exactly matches the table creation statement!
